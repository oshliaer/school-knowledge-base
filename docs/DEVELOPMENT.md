# Разработка

## Первый запуск

```bash
python -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## Команды

| Команда | Описание |
|---------|----------|
| `.venv/bin/python scripts/md2anki.py` | Собрать все Markdown-колоды |
| `.venv/bin/python scripts/md2anki.py "Папка предмета"` | Собрать одну Markdown-колоду (аргумент — путь/имя директории предмета, напр. `"Вероятность и статистика 7-9"`) |
| `.venv/bin/python scripts/md2anki.py --output /tmp/anki` | Указать выходную директорию |
| `.venv/bin/python scripts/fetch_countries.py` | Обновить данные стран из Wikidata (по умолчанию пишет в `География мира/countries.csv`; путь переопределяется флагом `--output`) |
| `.venv/bin/python scripts/fetch_flags.py` | Скачать/обновить флаги в `flags/` и обновить маппинг `flags.csv` (`wikidata_id → *_flag_file`), который затем читает `csv2anki.py` |
| `.venv/bin/python scripts/csv2anki.py` | Собрать CSV-колоды (например, «География мира») |

Результат сборки — `dist/<slug-названия>.apkg`: имя получается из `deck.yaml:name` через `slugify` (строчные латинские буквы и дефисы). Например, «Вероятность и статистика 7-9» → `dist/veroiatnost-i-statistika-7-9.apkg`.

## Формат карточек

YAML frontmatter задаёт тип и теги для всего файла. Карточки разделяются строкой `---` (с пустыми строками вокруг).

Карточки типов `basic` и `basic-reversed` разделяют лицевую и обратную стороны строкой `^^^` (с пустыми строками вокруг). Карточка без `^^^` пропускается при сборке. Для `cloze` разделитель не нужен.

### Basic (вопрос / ответ)

```markdown
---
type: basic
tags: [тег1, тег2]
---

id: 01-тема-slug
## Текст вопроса

^^^

Текст ответа.

---

id: 01-тема-slug2
## Следующий вопрос

^^^

Следующий ответ.
```

### Basic-reversed (двусторонняя)

```markdown
---
type: basic-reversed
tags: [термины]
---

id: 01-тема-slug
## Термин

^^^

Определение термина.
```

Создаёт две карточки: термин→определение и определение→термин.

### Cloze (пропуск в тексте)

```markdown
---
type: cloze
tags: [формулы]
---

Площадь круга равна {{c1::π·r²}}.

---

{{c1::Медиана}} делит упорядоченный ряд пополам.
```

Тип `cloze` определяется автоматически при наличии `{{c1::...}}` в тексте.

## Правила создания карточек

- Каждая карточка должна иметь поле `id:` перед заголовком `##`. Формат: `{номер файла}-{тема}-{slug}` (например, `01-таблицы-что-такое`). Slug, как и имя файла, пишется на русском. Без `id:` Anki теряет прогресс при пересборке.
- Тип `basic` — для понятий и вопросов. Тип `cloze` — для формул и расчётов.
- Если для параграфа нужны разные типы — создавать два файла: `01-тема.md` (basic) и `01-тема-задачи.md` (cloze).
- Имена файлов — на русском языке, без транслитерации (пример: `01-таблицы.md`, не `01-tablitsy.md`).

## deck.yaml

```yaml
id: 1748392650123456   # стабильный ID — не менять после первого импорта в Anki!
name: "Название колоды"
```

ID генерируется один раз. Если изменить — Anki создаст дубликат колоды.

## CI/CD

- **Push в `master`** → сборка + артефакт в GitHub Actions
- **Тег `v*`** → сборка + создание GitHub Release с `.apkg`-файлами
