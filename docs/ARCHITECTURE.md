# Архитектура

## Суть проекта

Публичный репозиторий с материалами для изучения школьных предметов. Исходники хранятся в репозитории, из них автоматически собираются файлы для импорта в Anki. Формат материалов может расширяться — не ограничивается Anki.

**Поток:** исходники в репо → `dist/*.apkg` → GitHub Release → пользователь

## Типы источников данных

В репозитории два независимых пайплайна. Каждый тип источника обрабатывается отдельным скриптом. Смешивать их **нельзя**.

| Тип | Источник | Скрипт | Пример |
|-----|----------|--------|--------|
| Markdown | `.md`-файлы с карточками | `md2anki.py` | Вероятность и статистика 7-9 |
| CSV | `countries.csv` + флаги | `csv2anki.py` | География мира |

`md2anki.py` не читает CSV. Добавлять поддержку CSV в `md2anki.py` не нужно.

## Структура репозитория

```
<Предмет>/                  # папка предмета (Markdown-колода)
  deck.yaml                 # имя колоды и стабильный Anki ID
  styles.css                # стили карточек (опционально)
  decks/
    01-тема.md              # карточки (basic/basic-reversed)
    01-тема-задачи.md       # карточки (cloze)

<Предмет CSV>/              # папка предмета (CSV-колода)
  deck.yaml
  styles.css                # CSS всегда отдельным файлом, не в скрипте
  countries.csv             # табличные данные (коммитятся в git)
  overrides.yaml            # ручные исправления данных
  flags/                    # кэш флагов (коммитятся в git)

scripts/
  md2anki.py               # Markdown → .apkg
  csv2anki.py              # CSV → .apkg
  fetch_countries.py       # Wikidata → countries.csv
  fetch_flags.py           # скачать флаги → flags/

docs/                      # документация для людей
dist/                      # выходные .apkg файлы (не коммитятся)
```

## Флаги

- Хранятся в `flags/` и коммитятся в git (SVG — предпочтительно, PNG — допустимо)
- `csv2anki.py` читает флаг из кэша по имени файла из CSV — сетевых запросов не делает
- Скачивание флагов — задача `fetch_flags.py`
- Данные (CSV) и флаги обновляются независимо
