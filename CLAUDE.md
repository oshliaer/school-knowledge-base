# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что такое этот проект

**Публичный репозиторий с материалами для изучения школьных предметов.**

Сейчас — карточки Anki. Формат материалов может расширяться. При принятии решений не ограничивай мышление только Anki — используй нейтральные формулировки ("учебные материалы", "материалы для изучения").

Основные пользователи — школьники и учителя. Техническая документация живёт в `docs/`. Этот файл — для AI-агентов.

## Назначение

Репозиторий хранит учебные материалы и собирает из них `.apkg`-файлы для импорта в Anki.

Поток: **исходники в репо → `dist/*.apkg` → GitHub Release → пользователь**

## Команды разработки

```bash
# Первый запуск — создать venv и установить зависимости
python -m venv .venv
.venv/bin/pip install -r requirements.txt

# Собрать все колоды
.venv/bin/python scripts/md2anki.py

# Собрать одну колоду
.venv/bin/python scripts/md2anki.py "Вероятность и статистика 7-9"

# Указать нестандартную папку вывода
.venv/bin/python scripts/md2anki.py --output /tmp/anki
```

Результат — `dist/<Название_колоды>.apkg`.

## Структура репозитория

```
<Предмет>/                  # папка предмета (любое название)
  deck.yaml                 # имя колоды и стабильный Anki ID
  styles.css                # стили карточек (опционально)
  decks/
    01-тема.md              # карточки темы (basic/basic-reversed)
    01-тема-задачи.md       # карточки темы (cloze), если нужны
    02-другая-тема.md
<Предмет CSV>/              # папка для колод на основе CSV (например, «География мира»)
  deck.yaml                 # имя колоды и стабильный Anki ID
  styles.css                # стили карточек — CSS всегда отдельным файлом, не в скрипте
  countries.csv             # табличные данные (коммитятся в git)
  overrides.yaml            # ручные исправления данных (коммитятся в git)
  flags/                    # кэш скачанных флагов (коммитятся в git)
scripts/
  md2anki.py               # конвертер md → .apkg (только для Markdown-колод)
  csv2anki.py              # конвертер CSV → .apkg (только для CSV-колод, отдельный скрипт)
  fetch_countries.py       # получение данных из Wikidata → countries.csv
  fetch_flags.py           # скачивание/обновление флагов → flags/ + обновление countries.csv
requirements.txt
.github/workflows/build.yml # CI/CD: сборка + публикация релиза
```

### Флаги

- Флаги хранятся в `flags/` и **коммитятся в git** (SVG — текст, PNG — допустимо)
- Текущее состояние: часть флагов в PNG (исторически), часть в SVG — постепенно заменяем PNG на SVG
- `csv2anki.py` читает флаг из кэша по имени файла из CSV (не скачивает, не делает сетевых запросов)
- Скачивание флагов — задача `fetch_flags.py`
- Для обновления конкретного флага: удалить файл из `flags/` и запустить `fetch_flags.py`
- Данные (CSV) и флаги обновляются независимо друг от друга

### Важно: CSV-колоды и Markdown-колоды — разные пайплайны

- `md2anki.py` **не читает CSV**. CSV-колоды обрабатываются только через `csv2anki.py`.
- Каждый тип источника данных (Markdown, CSV и т.д.) имеет **свой отдельный скрипт**.
- Добавлять поддержку CSV в `md2anki.py` **не нужно**.

### Команды для CSV-колод (география)

```bash
# Обновить данные из Wikidata (запускается вручную, результат коммитится)
.venv/bin/python scripts/fetch_countries.py

# Скачать/обновить флаги и проставить *_flag_file в CSV
.venv/bin/python scripts/fetch_flags.py

# Собрать колоду из CSV
.venv/bin/python scripts/csv2anki.py
```

## Формат карточек (`cards.md`)

YAML frontmatter задаёт тип и теги для всего файла.
Карточки разделяются строкой `---` (с пустыми строками вокруг).

Все карточки с двумя частями (`basic`, `basic-reversed`) разделяют лицевую и обратную стороны строкой `^^^` (с пустыми строками вокруг). Карточка без `^^^` считается некорректной и пропускается при сборке. Для `cloze` разделитель не нужен — у него одна часть.

### Basic (вопрос / ответ)

```markdown
---
type: basic
tags: [тег1, тег2]
---

id: xx-тема-slug
## Текст вопроса (лицевая сторона)

^^^

Текст ответа (обратная сторона).
Может быть **несколько абзацев**.

---

id: xx-тема-slug2
## Следующий вопрос

^^^

Следующий ответ.
```

### Basic-reversed (двусторонняя карточка)

```markdown
---
type: basic-reversed
tags: [термины]
---

id: xx-тема-slug
## Термин

^^^

Определение термина.
```

Создаёт две карточки: термин→определение и определение→термин.

### Cloze (пропуск в тексте)

```markdown
---
type: cloze
tags: [формулы]
---

Площадь круга равна {{c1::π·r²}}.

---

{{c1::Медиана}} делит упорядоченный ряд пополам.
```

Тип `cloze` также определяется автоматически при наличии `{{c1::...}}` в тексте.

## Правила создания карточек

- Каждая карточка должна иметь поле `id:` перед заголовком `##`. Рекомендуемый формат: `{номер файла}-{тема}-{slug}` (например, `01-tabl-chto-takoe`). Без `id:` Anki теряет прогресс при пересборке.
- В тексте карточек активно использовать выразительное форматирование для выделения определений и ключевых слов — это помогает при запоминании.
- **Не более 5–7 карточек на параграф** (только для «Вероятность и статистика 7-9»). Учебник содержит 45+ параграфов — экономить важно.
- **Не использовать ссылки на изображения** (`![](images/...)`) в «Вероятность и статистика 7-9». Вместо скриншотов таблиц использовать HTML-разметку `<table>` прямо в тексте карточки.
- Тип `basic` — для понятий и вопросов. Тип `cloze` — для формул и расчётов.
- Если для параграфа нужны разные типы — создавать два файла: `01-тема.md` (basic) и `01-тема-задачи.md` (cloze).
- **Имена файлов — на русском языке**, без транслитерации (пример: `01-таблицы.md`, не `01-tablitsy.md`).

## deck.yaml

```yaml
id: 1748392650123456   # стабильный ID — не менять после первого импорта в Anki!
name: "Название колоды"
```

ID генерируется один раз. Если изменить — Anki создаст дубликат колоды.
